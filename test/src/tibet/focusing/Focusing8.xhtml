Focus Test #8




This test sets up 4 focusable elements in multiple nested groups.
The 'goo' group has a 'focuswrap' attribute.
Field #7 (in the 'boo' group) has a tabindex value of '1' and field #8 (in the 'foo' group) has a tabindex value of '2'. And field #3 (not in any group) also has a tabindex value of '2'. Note that, in TIBET, tabindex values are processed 'per group', except upon page load. This means that the first focused field upon page load will be field #3. But when the 'goo' group is entered, field #8 will be focused and when the 'boo' group is entered, field #7 will be focused.
By pressing various keys, the focus can be navigated to them:

Home key navigates to the first field *in the first group*.
End key navigates to the last field *in the last group*.
Tab key navigates 'forward' to the next field and will *not* wrap around *at the group level* on the 'foo' group (since it has no 'focuswrap' attribute), but will wrap around on the 'goo' group (since it has a 'focuswrap' attribute).
Shift-Tab key navigates 'backward' to the previous field and will *not* wrap around *at the group level* on the 'foo' group (since it has no 'focuswrap' attribute), but will wrap around on the 'goo' group (since it has a 'focuswrap' attribute)
Ctrl-Tab key navigates 'forward' to the next field *in the same or next group* and will *ignore* any 'focuswrap' attributes.
Shift-Ctrl-Tab key navigates 'backward' to the previous field *in the same or last group* and will *ignore* any 'focuswrap' attributes.
PageUp key navigates to the first field *in the same group*.
PageDown key navigates to the last field *in the same group*.
Ctrl-PageUp key navigates to the first field *in the previous group* (ignoring any 'focuswrap' attributes).
Ctrl-PageDown key navigates to the first field *in the next group* (ignoring any 'focuswrap' attributes).
