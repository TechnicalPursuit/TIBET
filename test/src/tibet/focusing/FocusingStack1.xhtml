Focus Stack #1




This test sets up 4 'input' elements, all at the <body> element level.
None of the fields has a 'tabindex' attribute greater than -1, meaning that they're all programmatically controlled.
By pressing various keys, the focus can be navigated to them:

Home key navigates to the first field.
End key navigates to the last field.
Tab key navigates 'forward' to the next field and will wrap around.
Shift-Tab key navigates 'backward' to the previous field and will wrap around.

Because they're all at the <body> level (not inside of a 'tibet:group'), navigation between these elements will automatically wrap.

Fields #1, #2 and #4 all share the same focusing context (which is the document's 'body' element). Field #3 is wrapped with a span that defines a different focusing context. Therefore, it will be 'push'ed onto the focus stack when navigated to from one of the other elements. If it is then navigated away from (pressing Tab, etc.) the previously focused element will be navigated to.
