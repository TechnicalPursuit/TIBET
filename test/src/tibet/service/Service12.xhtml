Service Test #12




This test sets up a <tibet:service> tag that services data at a particular URI. It shows how to perform a standard HTTP <form> multipart related POST using no parameter, but having 2 bodies (defined using <tibet:data< tags in the head). It doesn't have a data type assigned, but will try to use a dynamically computed content type to hold the result data based on the data returned.
A variety of signals are thrown by the various objects involved.
